Summary

Gnome Sort also known as Stupid Sort is similar to Insertion Sort. The sort is simple to implement as it does not require any nested loops. The sort goes through the elements of the array in order. If it finds an element that is smaller than the element before it it swaps the element downwards until it is correctly placed(unlike insertion sort which waits to place an element until it's found the right position). Then it continues with the next element in the sequence. This like Insertion Sort is a stable sorting algorithm that splits the list into a sorted and unsorted section. Each element is added to the sorted section until the entire array becomes sorted. It runs in O(n2) but runs closer to O(n) for an almost sorted list.

Comparing Efficency Among Orderings

From these sets it's easy to see that if an array is in order or a few elements are out of place Gnome Sort is able to quickly sort the array. However, if a lot of elements are out of place or the array is reversed it takes much longer. A reversed array takes the max time possible as every new element has to be moved to the beginning of the sorted section of the array.
