Summary

Stooge Sort is one of the most inefficent sorts available that will still finish within a relatively reasonable time on small datasets(in comparison to sorts that are actively trying to not finish like some of the Bogosort Family). It has a time complexity of O(nlog 3 / log 1.5) or O(n2.7095...) this is substantially worse than even sorts like Selection and Bubble Sort that are expected to be poor.

Stooge Sort is a recursive sort that works by sorting the first 2/3rds of a list, the last 2/3rds of the list, and then the first 2/3rds again. It splits the sections recursively until they contain either 1 or 2 points and sorts the two point sections.

Comparing Efficency Among Orderings

As with some similar sorts(Selection and Cycle for example). The order of the data is irrelevant for Stooge Sort. This is because it will always compare the elements in the same pattern regardless.
