Summary

Insertion Sort is one of the most efficent O(n2) sorts. It's efficency can also be represented as O(n*k) where k is the max number of positions that an element is from it's correct position. Because of this it is extremely efficent for mostly ordered arrays.

Insertion sort works by separating the array into a sorted and unsorted portion. The sorted portion is at the beginning of the array. For each element in the array, the sort traverses backwards if the element before is larger than the element it copies that element into the current position and moves back again. If the previous element is smaller it places the element in the position. This means that traveling down elements are copied into the next location, but the element being inserted is only inserted one time. This is as opposed to Gnome Sort where each comparison leads to a swap if items are out of order.

Insertion sort also can be online. Meaning that it can sort while it is recieving a list. This is because it only needs access the sorted portionof the array and the next element for each round.
