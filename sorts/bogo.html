Summary

Bogosort is a highly ineffective sort. It's mostly a joke sort. The idea is that you take an array and shuffle then check if it's in order. If it is then you're done, if not shuffle it again. This is one of many sorts that have been created to be purposefully ineffecient while still being correct.

Versions

There are a lot of "versions" of Bogosort. Wikipedia lists Permutation Sort, Stupid Sort, Slowsort, Shotgun Sort or Monkey Sort as other names for Bogosort. It also lists Gorosort, Bogobogosort, Bozosort, and Worstsort as related algorithms.

Bozosort works like Bogosort, but it only swaps two random elements at time. This seems like it might work better as it won't switch the array from almost sorted to completely unsorted in one move. However, as it still is choosing to swap elements randomly it could still go on forever.

This is an optimization of Bozosort. Although, it maybe strange to optimize a sort that is expected to be terrible. It chooses a random pair of elements and swaps them only if they are out of order. While this is potentially more efficent. The random choices mean it's still less efficent than other sorts.

Stupid Sort was the original name of Gnome Sort. However, it has also become an alternative name for Bogosort.

Slowsort is in fact another sorting algorithm. So while Bogo is sometimes called that it is in fact a misnomer.

Gorosort comes from a Google Code Jam. The idea being that Goro can hold down some numbers and hit the table to shuffle the rest of the numbers. He chooses what to hold down optimally. Also, he has 4 arms.

Worstsort is an algorithm based on Badsort. It is guarenteed to complete if the universe does not end first.

Shotgun Sort is another name for Bogosort. Monkey Sort has implementations that match both Bogo, Permutation and Bozosort.

Permutation Sort is a version of Bogosort that removes repeats. Instead of shuffling between rounds it will try every permutation of the array. This means that it works in time O(n!) with a best case of O(n). It is faster if the later elements in the array are out of order in this implementation as it alters the permutation starting at the end.

Bogobogosort is a sort build not to succeed before the 'heat death of the universe'. This is a recursive Bogosort. Basically it starts with the first element and then adds one element at a time. If the new element is higher than the element before it, it will move on. If not it will shuffle everything up to and including that new element then start over at the first element again.

Quantum Bogosort sorts the list in O(1). It works as follows. Randomize the list so that you can't know the order without observing it. This divides the universe into O(n!) universes. This is fine due to the fact that all these universes exist anyway(Many-worlds interpretation). If the list is not sorted destroy the universe(this is an exercise for the reader). All remaining universes contain a list which is sorted. It should be noted this is not by default a stable sort.

from random import shuffle 

def bogo(array): 
    # check if the array is sorted. 
    # if every number is smaller than the next number it's sorted. 
    def isSorted(array): 
        for index, num in enumerate(array[:-1]): 
            if num > array[index + 1]: 
                return False 
        return True 
    # if the array is sorted return it 
    if isSorted(array): 
        return array 
    else: 
        # if the array is not sorted shuffle it and start over. 
        shuffle(array) 
        return bogo(array)
